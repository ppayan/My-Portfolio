Canvas Api Chatbot

A chatbot to navigate Canvas app.

Team: Phillip Payan & Aidden Torres | Duration: Fall 2025 Semester | Course: CSCI 4250 (Human Centered Computing)

Motivation

Why we built this

Students spend significant time navigating Canvas's multi-level menu system to access basic information like upcoming assignments, current grades, and course announcements. A typical query requires clicking through 3-5 different pages and waiting for each to load. This inefficiency becomes particularly problematic when students need to quickly check information between classes or before deadlines.

Problem Statement: Canvas LMS requires multiple navigation steps for simple information queries, creating friction that wastes student time and increases the likelihood of missing important updates.

Target Users: College students who need quick, frequent access to Canvas information without the overhead of web browser navigation.

Project Goals:

  • Reduce Canvas navigation time by 70% through conversational queries
  • Enable instant access to assignments, grades, and announcements via natural language
  • Implement secure local storage of API tokens with encryption
  • Provide desktop notifications for upcoming deadlines and new announcements

Online Study

Competitive Analysis & User Research

To understand user needs and identify pain points with existing solutions, we conducted a comprehensive competitive analysis of similar applications. We developed a Python-based web scraper to collect and analyze user reviews from both the Apple App Store and Google Play Store for five competing applications: Canvas Student, Google Classroom, Socratic by Google, Pi AI, and Notion. This automated approach allowed us to gather authentic user feedback at scale and identify common frustrations and desired features.

Research Methods:

  • Automated Review Scraping: Developed Python scraper using iTunes RSS feeds and Google Play API to collect up to 100 reviews per app per platform (~[TOTAL NUMBER] reviews total)
  • Sentiment Analysis: Implemented VADER sentiment analysis to classify reviews as positive, negative, or neutral
  • Topic Modeling: Applied Latent Dirichlet Allocation (LDA) to identify recurring themes and topics across reviews
  • Pain Point Analysis: Extracted and categorized user complaints from low-rated reviews (1-2 stars)
  • Competitive Analysis: Compared features, ratings, and user satisfaction across LMS apps, chatbot apps, and productivity tools

Tools & Technologies Used:

  • Python (pandas, requests, google-play-scraper)
  • VADER Sentiment Analysis
  • scikit-learn (TF-IDF, LDA topic modeling)
  • NLTK (natural language processing)
  • Data visualization (matplotlib, wordcloud)

Key Findings:

  • Navigation Complexity: 30.8% of negative reviews mentioned difficulty finding information within Canvas and similar LMS platforms, with users spending excessive time clicking through multiple menus
  • Grade Access Frustration: Students frequently complained about the multi-step process to check grades, with 28% of reviews mentioning "confusing" or "hard to find" grade information
  • Assignment Tracking Issues: 43% of users expressed frustration with keeping track of deadlines across multiple courses, citing poor notification systems and cluttered interfaces
  • Positive Chatbot Reception: AI chatbot apps (Socratic, Pi) received 64.8% average positive sentiment scores, indicating user acceptance of conversational interfaces for information retrieval
  • Quick Access Demand: Analysis of review keywords revealed strong demand for instant information access, with navigation complexity being the most frequently cited issue

Sentiment Analysis Results:

  • Overall sentiment distribution: 57.4% positive, 26.8% neutral, 15.8% negative (averaged across all apps)
  • Canvas Student: 2.79/5.0 average rating across platforms (53.7% positive sentiment)
  • Google Classroom: 2.21/5.0 average rating across platforms (34.8% positive sentiment)
  • Socratic by Google: 3.94/5.0 average rating (61.7% positive sentiment)
  • Pi AI: 3.06/5.0 average rating (67.9% positive sentiment)
  • Notion: 3.52/5.0 average rating (68.6% positive sentiment)
  • iOS vs Android: Android users reported 8.7% higher average satisfaction (3.26 vs 2.99 stars)

Identified Pain Points from 367 Negative Reviews:

  • Missing Features: 43 mentions (11.7%) - requests for features like chatbot assistance, better text input, and improved assignment submission
  • Sync Issues: 39 mentions (10.6%) - complaints about information not updating in real-time, app crashes after iOS updates
  • Slow Performance: 32 mentions (8.7%) - lag, crashes, or slow loading times
  • Complex Navigation: 30 mentions (8.2%) - confusing layouts or difficulty locating features
  • Login Problems: 7 mentions (1.9%) - authentication difficulties and "item not available offline" errors

Topic Modeling Insights:

LDA analysis revealed five main topic clusters across all reviews:

  • Topic 1: Assignment management and deadline tracking
  • Topic 2: Grade checking and GPA calculations
  • Topic 3: Navigation and user interface design
  • Topic 4: Notifications and reminders
  • Topic 5: Cross-platform sync and accessibility

Design Implications:

Based on this analysis, we identified the following requirements for our Canvas chatbot:

  • Conversational natural language interface to eliminate menu navigation
  • Instant grade and assignment access through simple queries
  • Proactive notifications and reminders for upcoming deadlines
  • Cross-platform desktop application for better performance than web-based solutions
  • Secure local API token storage to address authentication concerns
